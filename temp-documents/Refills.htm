Accessible patterns and components built with Bourbon and Bitters.

Maintained by thoughtbot, instructions at GitHub.

Refills are a starting point for interactive patterns and components that are accessible and semantic. If you're using the Bourbon suite, we encourage you to copy and paste them into your project's codebase to build from.

There may be minor variable or mixin conflicts with your own project. The components below were have been updated to work with Bourbon 5.0.0.beta.7 and Bitters 1.5.0.

  • Modal

    Modals, or generic dialogs, are used to overlay content for progressive disclosure.

    This implementation focuses into the modal when opened, and refocuses on the open button once closed, allowing them to be separated in the DOM without losing place or context. The modal can be closed on Escape, or clicking outside of it.

    Further enancements should hide the rest of the body's content for screen readers when a modal is open, and "trap" tabbable keyboard focus within the modal. A plugin like jquery-keyboard-trap can help achieve the latter.

    References: eBay Mind patterns, A11y Dialog.

  • Tabs

    Tabs are used to switch the display of local content without a page refresh.

    In this implementation, we wire up the association between the tab link and the tabpanel by an href and id. JavaScript sets up the appropriate ARIA roles. Left and right arrow keys can change focus between tabs, and pressing the tab key focuses the first element inside the tabpanel.

    References: Practical ARIA Examples

  • Tooltipped

    Tooltips are used to add descriptive text to interactive elements which are otherwise lacking descriptive text, like icon links and buttons that are only represented graphically.

    Tooltipped elements are not JavaScript dependent, because we don't need to update state for ARIA. We are instead able to toggle their display with CSS on the element's hover or focus states.
